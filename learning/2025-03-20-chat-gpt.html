Large Language Models (LLMs) and Their Role in the Livingston Labs Innovation Day Project

Introduction

Large Language Models (LLMs) are a relatively new type of technology that play a significant role in our Innovation Day project. Before giving our main presentation, we thought it would be helpful to spend some time clarifying how LLMs work and how we are using them in this project.

The Relationship Between LLMs and ChatGPT

When you hear the term “large language model,” it’s natural to think of something like ChatGPT. However, one key point is that ChatGPT itself is not just a large language model. ChatGPT is a web application that provides a user-friendly interface for interacting with a family of large language models (for example, GPT-4, GPT-4 Mini, etc.).

In other words, ChatGPT consists of both:

  1. A front end – the user interface that makes it feel like a chatbot or conversation window.
  2. A back end – the actual large language models that process and generate responses.

This conversational, back-and-forth style is only one use case for large language models—albeit a complex one that has garnered a lot of public interest.

Other Use Cases for Large Language Models

Beyond conversation, there are many other tasks LLMs can handle. Unlike a multi-turn chat, these tasks can sometimes be accomplished in a single “request → response” flow. Two examples include:

  1. Text Summarization
    LLMs can summarize large amounts of text. A typical prompt might be:
    “Please summarize the following text:”
    followed by the text to be summarized.

  2. Question-Related Retrieval
    In this scenario, the LLM answers a specific question using information you provide. A prompt might look like:
    *“You will be provided with:

    1. A question
    2. Information that can be used to answer that question.
      Please answer the question using only the information provided.”*

How We Are Using LLMs in Our Project

Our project focuses less on “chatting” with an LLM and more on tasks resembling a single request-and-response interaction. Because of this, we don’t need to rely on a web application interface; instead, we can send requests directly to a large language model via an API.

In particular, we are interested in question-related retrieval—having an LLM answer questions accurately by using a specific body of information. This approach aligns more closely with our project goals than simply conducting open-ended conversations with a large language model.
