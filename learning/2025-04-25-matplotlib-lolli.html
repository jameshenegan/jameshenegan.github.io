import pandas as pd
import matplotlib.pyplot as plt
import numpy as np

# --- 1.  point to the ‚Äú0/1‚Äù columns you care about -----------------------
indicator_cols = [c for c in df.columns if df[c].dropna().isin([0, 1]).all()]

# --- 2.  compute prevalence and sort descending --------------------------
means = df[indicator_cols].mean().sort_values(ascending=False)   # Series

# --- 3.  make a horizontal lollipop plot --------------------------------
n          = len(means)
fig_height = 0.45 * n + 1          # auto-size so labels never feel cramped
fig, ax    = plt.subplots(figsize=(6, fig_height))

y = np.arange(n)                                   # 0 ‚Ä¶ n-1
ax.hlines(y, 0, means.values, lw=2)                # stems
ax.scatter(means.values, y, s=50)                  # ‚Äúheads‚Äù

# y-axis: column names
ax.set_yticks(y)
ax.set_yticklabels(means.index)

# annotate with the mean (here shown as %)
for yi, m in zip(y, means.values):
    ax.text(m + 0.01, yi, f"{m:.1%}", va="center", fontsize=8)

ax.set_xlabel("Proportion (column mean)")
ax.set_xlim(0, means.values.max()*1.1)             # a little right margin
ax.invert_yaxis()                                  # largest on top
fig.tight_layout()
plt.show()

How it works üìå

Step Why
Compute df.mean() For 0/1 columns the mean is the proportion of 1‚Äôs.
sort_values(ascending=False) Gives you the ranking you want; we flip the y-axis later so the biggest sits on top.
Lollipop mechanics A horizontal line (hlines) from 0 ‚Üí mean plus a marker (scatter) at the right end.
Dynamic figure height 0.45 in √ó n keeps labels readable whether you have 5 or 50 features.
invert_yaxis() Makes the plot read ‚Äúhighest at the top.‚Äù
Inline text ax.text() places the percentage just beyond each marker. Tweak the 0.01 offset if your values are tiny.

Why cramming 50 items is tricky

  1. Physical real-estate ‚Äì a notebook cell is ~6‚Äì7 in wide.
    With 50 y-tick labels you need at least 50 √ó 0.30 in ‚âà 15 in of height just to keep 9- pt text from colliding.
  2. Export medium ‚Äì if it will live inside a PDF, you can let the canvas be tall and trust the viewer‚Äôs scroll bar; in a slide you can‚Äôt.
  3. Legibility vs.¬†density ‚Äì shrinking fonts and markers buys space, but below ~7 pt users start zooming.

Three practical ways to ‚Äúfit‚Äù 50 lollipops

Strategy When to use How to do it
Make the figure very tall Notebook, report, web page where scrolling is OK Scale height with n: fig, ax = plt.subplots(figsize=(6, 0.30*n + 1)), drop marker size (s=25), use fontsize=7. Save as PDF/SVG so vector text stays sharp even when readers zoom.
Split into columns / panels Slide decks, printed pages Use two or three axes laid out with matplotlib.gridspec or subplots(ncols=2). Slice the sorted series: left = means.iloc[:25], right = means.iloc[25:]. Each panel now has 25 items, so a ~10 in tall slide is enough.
Go interactive Dashboards, exploratory work With Plotly/Altair: make a vertical bar or lollipop, set height=800, and let the chart‚Äôs own scroll bar handle overflow‚Äîno need to squeeze anything. Tooltips replace the inline text labels, further reducing clutter.

Minimal Matplotlib pattern (auto-height)

import matplotlib.pyplot as plt
import numpy as np

means = df[indicator_cols].mean().sort_values(ascending=False)

n          = len(means)                     # e.g. 50
fig_height = 0.30 * n + 1                  # tweak the 0.30 ‚Ä≥ coefficient
fig, ax    = plt.subplots(figsize=(6, fig_height))

y = np.arange(n)
ax.hlines(y, 0, means.values, lw=1)
ax.scatter(means.values, y, s=25)          # small markers

ax.set_yticks(y)
ax.set_yticklabels(means.index, fontsize=7)  # small font
for yi, m in zip(y, means.values):
    ax.text(m + 0.005, yi, f"{m:.1%}", va="center", fontsize=6)

ax.set_xlabel("Proportion", fontsize=9)
ax.invert_yaxis()
fig.tight_layout()

fig.savefig("lollipop_50.pdf", bbox_inches="tight")  # vector!

Result: a ~16 inch-tall PDF that scrolls gracefully in a notebook and prints cleanly.

Bottom line

  • Slide / poster ‚ûú Break into two (or three) stacked or side-by-side panels.
  • Dashboard ‚ûú Use an interactive library and give users a scroll bar or hover tooltips.

Pick the route that matches where the figure will ultimately live; the code above covers the ‚Äúone-tall-figure‚Äù case, and you can adapt it to multi-panel layouts by slicing the sorted series before plotting.
