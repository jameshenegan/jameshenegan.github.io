Epic: End-to-End MLOps Project

Objective:
Transition from a single Azure ML workspace to a more robust, multi-workspace, version-controlled, and automated environment. This will improve collaboration, maintain model quality, and streamline the journey from development to production.

Feature 1: Versioned Code in an Azure DevOps Repository

Goal:
Establish a source-controlled codebase that integrates with Azure Machine Learning and supports best practices (e.g., branching strategies, code reviews, and a clear repository structure).

User Stories

  1. Create or Identify a Test Repository
    As a data scientist, I want to know which Azure DevOps repository to use so that I know where to store my code.

  2. Establish a Well-Structured Repository
    As a data scientist, I want the repository to have a clear and organized structure so that team members can quickly locate resources and code.

  3. Clone the Repository into the AML Development Workspace
    As a data scientist, I want to be able to clone the Azure DevOps repository into the notebooks section of the AML development workspace to facilitate development.

  4. Push Changes from the AML Environment to the Repository
    As a data scientist, I want to be able to push changes from my Azure ML development workspace back to the Azure DevOps repository so that all work is versioned.

  5. Ensure Git Is Installed Locally
    As a data scientist, I want Git installed on my local computer so I can develop code locally when needed.

  6. Implement Branching Strategies and Approval Gates
    As a data scientist, I want to use trunk-based development with proper approval gates so that only reviewed and accepted changes are merged into production.

  7. Store Production Code as Scripts
    As a data scientist, I want production code to be stored in scripts rather than notebooks for improved maintainability and automation.

Feature 2: ML Model Trained on Simulated Data

Goal:
Develop an ML pipeline that uses simulated data to train and evaluate machine learning models.

User Stories

  1. Incorporate Simulated Data into the ML Pipeline
    As a data scientist, I want to integrate simulated data into my ML pipeline so that I can train and test models even in the absence of real-world data.

Example Work Items

  • Task: Develop scripts or notebooks to simulate data based on project requirements.
  • Task: Integrate the simulated data generator into the model training pipeline.
  • Task: Validate the quality and consistency of the simulated data.

Feature 3: Separate AML Environments

Goal:
Separate development and production environments to ensure secure, isolated workflows and adhere to best practices.

User Stories

  1. Create a Production AML Workspace
    As a data scientist, I want the production AML workspace to be separate from the development workspace so that production remains free of experimental work and maintains security best practices.

  2. Move Trained Models to Production
    As a data scientist, I want to be able to move trained models from the development AML workspace to the production workspace so that inferencing occurs in a stable, secure environment.

  3. Perform Inferencing in Production
    As a data scientist, I want inferencing to be executed in the production workspace so that results can be reliably delivered to downstream business processes.

Example Work Items

  • Task: Provision and configure separate AML workspaces for development and production.
  • Task: Automate the transfer of trained models from development to production.
  • Task: Set up inferencing services in the production workspace.

Feature 4: Continuous Integration and Continuous Deployment (CI/CD)

Goal:
Implement automated pipelines to ensure that code changes, model training, testing, and deployment occur in a reliable, repeatable, and traceable manner.

User Stories

  1. Establish a Code Review Process
    As a data scientist, I want all code changes to be reviewed before merging into the main branch to maintain high-quality code standards.

  2. Implement Automated Testing
    As a data scientist, I want automated tests (unit, integration, and performance tests) to run on new code so that issues are caught early and the system remains stable.

  3. Automate Model Retraining
    As a data scientist, I want the ML model to be retrained automatically whenever new code is merged so that it stays updated with the latest changes.

  4. Perform Automated Model Checks
    As a data scientist, I want the newly trained model to undergo automated performance and quality checks so that I can be confident in its effectiveness before deployment.

  5. Set Up Approval Gates for Deployment
    As a manager, I want an approval gate in place to ensure that a new model is only deployed to production after necessary reviews and validations.

  6. Implement Automated Deployment
    As a data scientist, I want the validated model to be automatically deployed to the production AML workspace once all tests pass and approvals are met, ensuring a seamless release process.

Example Work Items

  • Task: Configure Azure DevOps pipelines for automated testing and code review integration.
  • Task: Develop and integrate automated tests into the CI/CD pipeline.
  • Task: Set up automated retraining and model performance evaluation steps in the pipeline.
  • Task: Create approval workflows within the deployment pipeline.
  • Task: Automate the deployment process to the production AML workspace.

Feature 5: Centralized Knowledge Base for MLOps Research

Goal:
Establish a single, accessible repository for documentation, insights, and best practices related to MLOps to support ongoing learning and collaboration.

User Stories

  1. Create a Centralized Knowledge Base
    As a data scientist, I want to create a centralized knowledge base so that I can easily store and share MLOps research, documentation, and insights.

Example Work Items

  • Task: Design the structure and taxonomy for organizing MLOps content.
  • Task: Migrate existing documentation and establish guidelines for future contributions.
