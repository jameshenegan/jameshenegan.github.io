Tutorial: Key Concepts in Machine Learning - Inferencing, Predictions, and Scoring

Welcome to this tutorial on some foundational terms in the machine learning lifecycle. This guide is structured to clarify each term’s role, helping you understand how they fit together in building, testing, and deploying a machine learning model.

Step 1: Training Your Model

What is Training?

Training is the first step in creating a machine learning model. Here, we feed the model a labeled dataset (where the outcome is known), so it can learn patterns and relationships within the data.

Goal of Training

The goal is for the model to understand how input data relates to an output label. For instance, in image classification, the model will learn features in images that correspond to specific categories (e.g., “cat” vs. “dog”).

Example

Imagine a model that predicts house prices based on features like square footage, location, and number of rooms. During training, the model analyzes the historical data, finding relationships between house features and their prices.

Step 2: Model Evaluation

What is Evaluation?

Evaluation tests the model’s performance by assessing how well it generalizes to data outside the training set. This step usually involves a separate dataset, often called a validation or test set, to simulate new, unseen data.

Why Evaluation Matters

Model evaluation tells us whether the model is likely to perform well on real-world data. Poor results indicate the model might be overfitting (too specific to the training data) or underfitting (too simplistic).

Example

Continuing with our house price model, we might evaluate it on a different set of houses to see if it accurately predicts their prices. Common evaluation metrics include accuracy, precision, recall, and mean squared error (for regression).

Step 3: Deploying the Model (Serving)

What is Serving?

Once you’re satisfied with the model’s performance, serving is the process of making the model accessible for real-time or batch predictions. This typically involves setting up the model as a microservice or endpoint that can receive data and return predictions.

Options for Serving

  • On-Device: Deploying the model on local devices (e.g., mobile apps).
  • On-Premises: Hosting within a controlled, local environment.
  • Cloud-Based: Deploying on cloud infrastructure, which is ideal for scalability and accessibility.

Step 4: Understanding Predictions

What is a Prediction?

Once the model is deployed, it can start generating predictions—the process of using the model to analyze new input data and produce an output.

There are two types of predictions:

  1. Online Prediction: Used for real-time predictions, often when quick response times are needed. For example, an online product recommendation system updates in real time as a user browses.
  2. Batch Prediction: Used for generating predictions on large datasets offline. This is ideal for cases like analyzing customer data overnight to prepare recommendations for the next day.

When to Use Prediction

Predictions can apply to different use cases:

  • Forecasting Future Outcomes: Predicting a value, such as the likelihood of customer churn or the demand for a product.
  • Classification Tasks: Assigning labels, like identifying spam emails or determining the sentiment of a text.

Step 5: Inferencing

What is Inferencing?

In inferencing, we use a trained model to generate predictions from new data. While the term “prediction” is often used for forecasting future values, inferencing can encompass broader applications, such as image or text classification where we aren’t predicting the future but are instead classifying or identifying patterns in the data.

Inferencing is a term you’ll see a lot in production machine learning systems, referring to using the model’s learned knowledge to process new data.

Example

In an image classification system, inferencing might involve analyzing a new image and determining that it contains a dog. This isn’t a “prediction” in the temporal sense but rather a classification based on the model’s learned understanding.

Step 6: Scoring

What is Scoring?

Scoring is the process of applying the trained model to data to produce a numerical score or classification. In some fields, particularly in finance and marketing, scoring is commonly used to assign a numeric value to a specific outcome.

Why Scoring is Useful

Scoring often applies when predictions need to be in the form of a score, risk level, or probability. This approach can be crucial for:

  • Risk Assessment: Assigning a credit risk score based on financial history.
  • Customer Segmentation: Scoring potential leads to determine marketing priority.
  • Ranking: Assigning relevance scores in recommendation systems.

Example

A credit scoring model might analyze a customer’s financial data and produce a risk score, indicating the likelihood they will repay a loan. In this context, “scoring” emphasizes the generation of a specific, actionable number.

Putting It All Together

The machine learning workflow typically follows a sequence of:

  1. Training → 2. Evaluation → 3. Serving → 4. Prediction/Inferencing/Scoring.

Each term reflects a distinct aspect of the model’s lifecycle, whether it’s learning from data, being evaluated, generating results, or being deployed for real-time or batch use.

In this tutorial, you’ve learned about the key stages in deploying a machine learning model and the purpose behind terms like inferencing, prediction, and scoring. Understanding these terms will clarify the goals and outcomes at each step of your machine learning projects.
