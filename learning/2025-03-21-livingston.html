Large Language Models (LLMs) and Their Role in the Livingston Labs Innovation Day Project

Introduction

Large Language Models (LLMs) are AI systems trained on vast amounts of text data, enabling them to understand and generate human-like text. They are considered “large” because they contain billions of parameters—tunable “knobs” that help these models learn patterns in language at scale.

Before giving our main presentation, we thought it would be helpful to spend some time clarifying how LLMs work and how we are using them in this project.

The Relationship Between LLMs and ChatGPT

When you hear the term “large language model,” it’s natural to think of something like ChatGPT. However, one key point is that ChatGPT itself is not just a large language model. ChatGPT is a web application that provides a user-friendly interface for interacting with a family of large language models (for example, GPT-4, GPT-4 Mini, etc.).

In other words, ChatGPT consists of both:

  1. A front end – the user interface that makes it feel like a chatbot or conversation window.
  2. A back end – the actual large language models that process and generate responses.

The “chat” format is just one way to interact with LLMs; in reality, any application can send requests to these models, and you’ll still get a response, even without a chatbot interface.

Other Use Cases for Large Language Models

Beyond conversation, there are many other tasks LLMs can handle. Unlike a multi-turn chat, these tasks can often be accomplished in a single “request → response” flow. Below are three examples:

  1. Text Summarization
    LLMs can summarize large amounts of text. A typical prompt might be:
    “Please summarize the following text:”
    followed by the text to be summarized.

  2. Question-Related Retrieval (Direct Answer)
    Here, the LLM answers a specific question using information you provide. A prompt might look like:
    *“You will be provided with:

    1. A question
    2. Information that can be used to answer that question.
      Please answer the question using only the information provided.”*

  3. Question-Related Retrieval (Finding Relevant Passages)
    Rather than generating a single, direct answer, the LLM can help locate relevant information within a large body of text. For example:
    “You will be given a question and a large amount of text. I want you to locate all information in the text that is relevant to the question.”

These examples illustrate that LLMs are versatile tools capable of performing different language-based tasks, whether it’s summarizing content, generating answers, or pinpointing relevant passages.

Why We Are Using LLMs in Our Project

Our Innovation Day project focuses less on “chatting” with an LLM and more on tasks resembling a single request-and-response interaction. Because of this, we don’t need to rely on a web application interface; instead, we can send requests directly to a large language model via an API.

In particular, we are interested in question-related retrieval—having an LLM locate and provide answers from a specific body of information. This approach helps ensure accuracy by grounding responses in real data, aligning more closely with our project goals than open-ended conversations might.

Conclusion and Next Steps

We hope this introduction clarifies how LLMs work and how they differ from chat-based interfaces. By understanding that ChatGPT is simply one application of large language models, you can begin to see how LLMs can be embedded into a wide range of products and services. With this foundation, we’ll now demonstrate how we’ve tailored these models for our specific use case at Livingston Labs—ensuring that they reliably retrieve and synthesize the information we need for innovation.
