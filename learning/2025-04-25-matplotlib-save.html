Here are the essentials you need 90 % of the time:

import matplotlib.pyplot as plt
import pathlib   # optional but handy

# 1️⃣  build the plot
fig, ax = plt.subplots(figsize=(4, 3))
ax.hist(data, bins=20)
ax.set_xlabel("Value")
ax.set_ylabel("Count")
ax.set_title("My histogram")

# 2️⃣  choose an output path
out_dir  = pathlib.Path("figures")      # guarantees cross-platform slashes
out_dir.mkdir(exist_ok=True)
outfile  = out_dir / "histogram.png"    # or .pdf / .svg / .eps / .jpg …

# 3️⃣  save it
fig.savefig(
    outfile,
    dpi=300,            # ⬅️  pixel density (ignored for vector formats)
    bbox_inches="tight" # ⬅️  trims extra whitespace
)
plt.close(fig)          # optional: frees memory in scripts / loops

Key parameters of Figure.savefig

Argument Typical values What it does
fname 'plot.png', Path("plots/plot.pdf"), BytesIO() Path-like, URL, or file-like object to write to.
dpi 72–100 for screen, 150–300 for slides, 300–600 for print Affects raster formats (png, jpg, tiff). Has no effect on vector (pdf, svg, eps).
format 'png', 'pdf', 'svg', 'jpg', 'tiff' Rarely needed—Matplotlib infers it from the file extension.
bbox_inches 'tight' or None 'tight' reduces whitespace padding; can slightly change canvas size.
transparent True / False PNG & PDF only. Makes background fully transparent—handy for slides.
facecolor / edgecolor color spec Override the default figure face/background.
pad_inches float (inches) Extra padding when using 'tight'.

Raster vs. vector

  • Raster (PNG, JPG, TIFF): fixed pixels. Large datasets render fast, but text/lines can blur when zoomed. Control sharpness with dpi.
  • Vector (PDF, SVG, EPS): based on geometric primitives. Infinitely sharp when zoomed or printed, but huge scatterplots can bloat file size.

Common patterns

Goal One-liner
Quick save to current directory plt.savefig("plot.png")
High-DPI PNG for print fig.savefig("plot.png", dpi=600, bbox_inches="tight")
Transparent background for slides fig.savefig("plot.pdf", transparent=True)
SVG for the web fig.savefig("plot.svg")
Auto-numbered files inside a loop fig.savefig(f"plots/hist_{i:02d}.png")

Pro tips

  1. Create the directory once (pathlib.Path.mkdir(exist_ok=True)), so your script never fails on a fresh machine.
  2. Close figures (plt.close(fig)) when batch-producing many images to avoid a memory leak or slowdowns.
  3. In notebooks, prefer fig.savefig(...) over plt.savefig(...)—it’s clearer which figure you’re saving.
  4. Vector first: when unsure, export PDF/SVG for archival and down-convert to PNG/JPG later.

That’s it—you now have the “save to file” workflow that scales from quick notebook snapshots to publication-quality figures.
