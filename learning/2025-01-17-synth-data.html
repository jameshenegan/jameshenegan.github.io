    • Centralized test automation servers
    • Containers (e.g., Docker)[1]

  2. G-Repository: This is a centralized and secure repository that manages configuration files, user access, and system usage. It exists on-premises within the corporate environment[3].

  3. GenRocket Multi-User Server (GMUS): This shared environment can be deployed on-premises to manage test data requests and delivery[1].

Cloud Capabilities

GenRocket also leverages cloud technology:

  1. GenRocket Cloud: This component is hosted on AWS and is used for test data design[3]. It provides:

    • A secure multitenant environment
    • Self-service components for licensed users
    • High availability and scalability

  2. Virtual Private Cloud (VPC): Most customers (90%) are hosted in a secure VPC environment on AWS[3].

  3. Dedicated Private Cloud (DPC): An optional offering for customers who require a dedicated cloud environment[3].

Hybrid Approach

GenRocket’s architecture is designed as a secure hybrid cloud environment:

  • Test data design takes place in the cloud
  • Test data generation occurs on-premises, behind the customer’s firewall[3]

This separation enhances security while allowing for scalability and ease of use. The G-Repository acts as a secure proxy, facilitating communication between on-premises components and GenRocket Cloud Services[3][4].

In conclusion, GenRocket offers a flexible solution that can work both on-premises and in the cloud, with a hybrid approach that leverages the strengths of both deployment models.
