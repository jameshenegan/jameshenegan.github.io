Below are two common ways to pin each cluster ID to an exact CSS-named color in Matplotlib. Pick whichever style feels more natural for the rest of your code.

1) Build the color list yourself (simplest)

import matplotlib.pyplot as plt

# x, y are the first two PCs; labels is an array of 0-3
# e.g., x = pcs[:, 0];  y = pcs[:, 1]

label2color = {
    0: "steelblue",     # any CSS4 name works
    1: "orangered",
    2: "mediumseagreen",
    3: "goldenrod"
}

colors = [label2color[l] for l in labels]

fig, ax = plt.subplots()
ax.scatter(x, y, c=colors, alpha=0.8, edgecolor="none")

# neat legend
handles = [
    plt.Line2D([], [], marker="o", linestyle="",
               color=label2color[k], label=f"Cluster {k}")
    for k in sorted(label2color)
]
ax.legend(handles=handles, title="Clusters")
ax.set_xlabel("PC 1"); ax.set_ylabel("PC 2")
plt.show()

Why this works: scatter accepts either a single color or a list of per-point colors. By turning the label array into that list, you have total control.

2) Use a fixed ListedColormap + BoundaryNorm

If you prefer passing the label array straight to c= and letting Matplotlib look up colors, wrap your CSS names in a ListedColormap and give a matching normalizer:

import matplotlib.pyplot as plt
import matplotlib.colors as mcolors
import numpy as np

css_colors = ["steelblue", "orangered", "mediumseagreen", "goldenrod"]
cmap = mcolors.ListedColormap(css_colors)

# one boundary between every integer label
norm = mcolors.BoundaryNorm(np.arange(-0.5, len(css_colors)+0.5), cmap.N)

fig, ax = plt.subplots()
sc = ax.scatter(x, y, c=labels, cmap=cmap, norm=norm, alpha=0.8, edgecolor="none")

# legend (same trick as above)
handles = [
    plt.Line2D([], [], marker="o", linestyle="",
               color=css_colors[i], label=f"Cluster {i}")
    for i in range(len(css_colors))
]
ax.legend(handles=handles, title="Clusters")
ax.set_xlabel("PC 1"); ax.set_ylabel("PC 2")
plt.show()

BoundaryNorm ensures that label 0 maps to the first entry in the colormap, label 1 to the second, and so on—no matter how many points belong to each cluster.

Tips & gotchas

Issue Fix
Matplotlib warns about edgecolors on unfilled markers either ignore (harmless) or add edgecolor="none"
Your labels start at 1 or aren’t consecutive make the dict keys (method 1) or the norm boundaries (method 2) match those exact integers
You want global color definitions put label2color or css_colors at module level and reuse across plots

Use whichever pattern keeps your code clearest; both give you full, explicit control over the mapping from cluster ID → color.
