Tutorial: Setting Up a .gitignore for MLflow Projects

When working on machine learning projects that use MLflow, it’s essential to ensure that only relevant files are tracked in your Git repository. Files like experiment logs, artifacts, virtual environments, and large datasets can clutter your Git history and increase repository size unnecessarily. In this tutorial, we’ll walk through creating a proper .gitignore file for your MLflow projects.

Step 1: Understanding What Needs to Be Ignored

Here are the typical types of files you’ll want to exclude:

  • MLflow logs and artifacts: MLflow generates logs and stores artifacts in the mlruns/ directory by default. You usually don’t need to track this in Git.
  • Model artifacts: Large model files (e.g., .pkl, .h5, .pt) should be excluded, as they can significantly increase repository size.
  • Virtual environments: If you’re using venv or conda, exclude the virtual environment directories (venv/ or env/) since these are machine-specific.
  • Temporary files: Jupyter notebook checkpoints, log files, and temporary files generated during runs should be excluded to avoid clutter.

Step 2: Creating the .gitignore File

  1. Open your project’s root directory in a terminal.

  2. Create a new .gitignore file, or open an existing one if you have it:

    touch .gitignore

  3. Add the following content to your .gitignore file:

    # Ignore MLflow logs and artifacts
mlruns/
artifacts/

# Python-specific ignores
__pycache__/
*.py[cod]
*.pyo
*.egg-info/
*.log
.env  # if using environment variables

# Virtual environments
venv/
env/

# Jupyter Notebook checkpoint files
.ipynb_checkpoints

# Ignore large datasets (optional)
data/
datasets/
*.csv  # Example for CSV data files

# Ignore model files
*.h5  # Keras model files
*.pkl  # Pickle files
*.pt  # PyTorch model files
*.joblib  # Scikit-learn joblib files

# Ignore Conda or pip environment files
conda.yaml
requirements.txt

# Ignore Docker-related files if using Docker for MLflow deployments
docker-compose.yml
Dockerfile

Step 3: Customize Based on Your Project

  • If your team collaborates on experiment tracking and needs to share the mlruns/ folder, you might consider not excluding it from Git.
  • For larger projects, you might need to add specific file formats or directories (e.g., *.parquet for large datasets).

Step 4: Verify .gitignore

After editing the .gitignore file, verify that the right files are being excluded by running:

git status

This command will show you which files are being tracked or ignored.

Recap:

  • You created a .gitignore file that excludes unnecessary files like MLflow logs, large model artifacts, and virtual environments.
  • This ensures your Git repository remains clean and efficient.
  • You can customize the .gitignore to fit your specific project needs.

This setup allows you to focus on the essential code and configurations in your repository while ignoring generated files that don’t need version control.
